웹 브라우저와 웹 서버의 관계
주소를 입력해서 요청을 하면 웹 서버는 그 요청에 따른 정보를 찾아서 응답해준다.

노드 js의 특징
어떤 코드를 넣느냐에 따라 사용자에게 전송하는 데이터가 바뀐다. - 프로그래밍 적이다.

프로그래밍할 때 생각해야 하는 것
어떠한 데이터가 있는가, 어떻게 처리하는가, 데이터의 종류, 그것을 처리하는 방법

리터럴
데이터를 타입을 표현한다.

url
1        2                    3     4     5           
http://opentutorials.org:3000/main?id=HTML&page=12
1. protocol  - 웹서버가 서로 데이터를 주고 받기 위해서 만든 통신 규칙
2. host(domain) - 어떤 특정한 인터넷에 연결되어 있는 컴퓨터를 가리키는 주소
3. port - 한 개의 컴퓨터에 여려개의 서버가 있을 경우가 있으므로 구별해주는 번호
4. path - 컴퓨터 안에 있는 어떤 디렉토리에 어떤 파일인지를 가리킨다.
5. query string - 읽고 싶은 데이터의 정보를 전달한다

모듈
기본적으로 제공하는 기능들을 그룹핑 해놓은 각각의 그룹들

CRUD(Create Read Update Delete)
정보를 다루는 핵심적인 처리 방법들

Parameter
입력되는 정보의 형식
Argument
형식에 맞게 실제로 입력한 값

배열
관리해야할 데이터가 방대할 경우 처리하기 쉽게 해주는 도구

함수의 중요성
1. 중복의 방지
2. 유지보수에 용이해짐
3. 가독성이 높아진다.

문법 
__dirname
파일의 경로를 나타내준다.

`
템플릿 리터럴의 시작과 끝을 나타낸다. 
문자열을 그대로 쓰고 ${변수}를 써주면 +없이 넣을 수 있다.

require(모듈이름)
모듈을 사용한다.

response.end(내용)
내용을 페이지에 띄운다.

readFile(파일,function(err,data)
해당 파일을 읽는다.

writeHead(200)
웹브라우저가 웹 서버에 접속했을 때 웹 서버가 응답함
200을 서버가 브라우저에게 주면 파일을 전송 성공
찾을 수 없을 경우는 404를 반환

readdir(폴더경로,function(error, filelist)
폴더의 파일을 배열 형태로 읽어들여라.(filelist)
